Uppgiftsbeskrivning till Moment 3

Jag har tittat på föreläsningarna om SASS och det verkar vara enekelt att använda men samtidigt ha många funktioner som man inte har i vanligt css. Jag hade aldrig jobbat med det själv innan den här momenten, så det var första gången.
I början trodde jag att det var svårt att komma igång med SCSS men det var det inte! jag tycker att det är enkelt att koda med SCSS och att det har mycket bättre struktur än CSS och man kan göra flera saker med det så jag hoppas att Webbläsarna stöder det i framtiden så att det blir enklare och använda det.
Jag kommer att använda det säkert i mina projekt i framtiden.
    I den här delen kommer jag att gå igenom fördelarna med SASS:
  • Med hjälp av partials kan man ha olika mindre filer så det blir enklare att läsa koden.
  • @extend, med hjälp av det behöver man inte upprepa en viss kodrad flera gånger så det blir mycket snabbare att koda!
  • Variabler, med hjälp av det nbehöver man inte upprepa egenskaper som färg, font-stiletc om och om igen därför blir allt smidigare!
  • Nesting, med den här funktionen kan man koda med bättre stuktur än vanligt CSS.
  • Det går att koda i SASS alltså man kan ha metoder, functions precis som JavaScript.
    I den här delen kommer jag att gå igenom nackdelarna med SASS:
  • Webbläsarna kan inte läsa SCSS filer därför måste filerna konverteras till vanlig CSS så att de kan läsas in
  • Svår felsökning!
